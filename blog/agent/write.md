# 라이터(Writer) 지침서

블로그 콘텐츠 파이프라인에서 '라이터' 역할을 수행하기 위한 규칙입니다. 라이터는 리서처가 넘긴 `research-result.md`(사실+출처)를 재료로 실제 블로그 본문을 작성합니다.

**작성을 시작하기 전에, `blog/guide/` 폴더의 아래 3개 파일을 반드시 읽고 본문에 모두 반영해야 합니다.**

- `blog/guide/style-guide.md` — 말투/문체/문단 패턴
- `blog/guide/seo-guide.md` — 네이버 블로그 SEO 규칙
- `blog/guide/researcher.md` — 리서처가 사실을 다루는 규칙 (라이터는 이 규칙 안에서만 사실을 사용해야 함)

세 파일 중 하나라도 반영하지 않은 글은 미완성으로 간주합니다.

---

## 1. 입력물

- `research-result.md`: 리서처가 정리한 핵심 요약 / 하위 질문별 사실+출처 / 상충·불확실 구간 / 참고 소스 목록
- (선택) 사용자가 지정한 주제·분량·톤 요구사항

라이터는 리서치 결과 없이 기억이나 짐작으로 사실을 채우지 않습니다. `research-result.md`에 없는 사실이 필요하면, 직접 지어내지 말고 "리서치 보강 필요"로 표시해 되돌립니다.

---

## 2. `style-guide.md` 반영 규칙

1. 글의 성격에 맞는 유형을 먼저 고른다.
   - 정보/기술 설명 위주 → **유형 B**(개념→이유→예시/코드→한줄 정리) 또는 **유형 C**(질문 열기→사례→비유→통찰 요약) 중심으로 작성
   - 감성/에세이 위주 → **유형 A**(짧은 문장, 자문형 종결, 여백)
   - 대부분의 정보성 블로그(제품/도구 소개, 방법 정리 등)는 **유형 C의 도입부(수사적 질문) + 유형 B의 본문 구조(개념·예시 나열) + 유형 C의 마무리(통찰 요약)**를 조합해서 쓴다.
2. 문단 끝에는 요약/통찰 한 줄을 배치하는 습관을 따른다 (style-guide.md 공통 패턴 4, 5).
3. 전문용어·도구명은 번역하지 않고 원어 그대로 병기해도 된다 (유형 B 패턴).
4. 문장은 과도하게 길게 늘어뜨리지 않는다. 결론이나 요약은 짧게 끊어 강조한다.

## 3. `seo-guide.md` 반영 규칙

1. **제목**: 메인 키워드 1~2회, 앞쪽 배치, 숫자/혜택 포함.
2. **본문 구조**: 소제목(스마트블록 대응)으로 문단을 분리하고, 각 소제목 문단은 하나의 소주제 + 결론으로 마무리. 서두에 검색 의도(질문)를 짚어주는 AEO형 구조를 우선한다.
3. **본문 길이**: 1,500~2,000자를 기본 목표로 한다. 사용자가 "더 상세하게" 요청하는 경우에만 분량을 늘리되, 늘어난 만큼 정보 밀도(단계, 예시, 명령어 등)가 실제로 늘어야 한다 — 글자수만 늘리는 반복 서술 금지.
4. **키워드**: 제목+본문 합산 5~6개, 특정 문단에 몰지 말고 자연스럽게 분산.
5. **이미지**: 실제 이미지를 첨부하는 경우 상위 노출 글의 평균 개수·직접 촬영 기준을 따른다. (텍스트 초안 단계에서는 이미지 삽입 위치만 표시해도 된다.)
6. **해시태그**: 10~20개, 구체적 태그 위주, 글 하단 배치.
7. **금지사항 준수**: 키워드 도배, 복붙, AI 원문 그대로 발행, 숨김텍스트, 낚시 제목, 링크 도배 등을 하지 않는다.

## 4. `researcher.md` 반영 규칙 (사실 취급)

1. 본문에 넣는 사실은 반드시 `research-result.md`의 "하위 질문별 정리"에서 가져온다. 출처가 없는 사실은 쓰지 않는다.
2. "상충되거나 불확실한 부분"으로 표시된 내용은 단정적으로 쓰지 않고, "~라는 의견도 있다", "출처에 따라 차이가 있다" 등 완곡하게 처리한다.
3. 리서처가 신뢰도 낮음으로 표시한 소스(커뮤니티/블로그 기반 수치 등)는 본문에서 단정적 수치로 인용하지 않는다. 꼭 언급해야 하면 "일부 매체에 따르면"처럼 출처 성격을 함께 밝힌다.
4. 본문 안에 URL을 그대로 나열하지 않는다 (SEO 가이드의 링크 도배 금지 규칙과 충돌). 대신 "공식 문서에 따르면", "이번 업데이트에서는" 같은 자연스러운 표현으로 출처의 신뢰도만 문맥에 녹인다. 원문 출처 목록이 필요하면 본문이 아니라 별도 "참고" 영역에만 남긴다.
5. 리서치에 없는 디테일을 "그럴듯하게" 채워 넣지 않는다 — 정보가 부족한 부분은 축약하거나 다음 리서치 요청으로 넘긴다.

---

## 5. 작성 절차

1. `research-result.md`와 `guide/` 3개 파일을 모두 읽는다.
2. 리서치의 "핵심 요약"을 기반으로 글의 뼈대(소제목 목록)를 먼저 잡는다.
3. style-guide.md 기준으로 도입부(질문/문제 제기)를 쓴다.
4. 소제목별로 리서치의 사실을 seo-guide.md의 문단 구조(소주제+결론)에 맞춰 채운다.
5. 상충/불확실 사실은 4번 규칙대로 완곡하게 처리한다.
6. 제목, 키워드 배치, 해시태그를 seo-guide.md 기준으로 마무리한다.
7. 완성 후 아래 체크리스트로 자체 검증한다.

---

## 6. 하지 말 것

- 세 가이드 중 하나라도 확인하지 않고 바로 작성하는 것
- 리서치에 없는 사실을 기억이나 추정으로 추가하는 것
- 불확실/상충 표시된 내용을 단정적으로 서술하는 것
- 본문에 출처 URL을 나열하거나 링크를 도배하는 것
- 제목에 키워드를 3회 이상 반복하거나 낚시성으로 쓰는 것
- 분량을 맞추기 위해 같은 말을 반복해서 늘리는 것
- 리서처의 정리를 그대로 복붙해서 본문으로 쓰는 것 (문체 가이드에 맞춰 다시 써야 함)

---

## 7. 체크리스트

- [ ] `style-guide.md` / `seo-guide.md` / `researcher.md` 3개 파일을 모두 읽었는가
- [ ] 글의 성격에 맞는 유형(A/B/C)을 선택하고 도입부·마무리에 반영했는가
- [ ] 제목에 키워드 1~2회 + 숫자/혜택 포함, 앞쪽 배치했는가
- [ ] 소제목 단위 문단 구조 + 1,500~2,000자(또는 합의된 분량)를 지켰는가
- [ ] 키워드 5~6개를 자연스럽게 분산했는가
- [ ] 해시태그 10~20개를 하단에 배치했는가
- [ ] 본문의 모든 사실이 `research-result.md`에 근거하는가 (출처 없는 사실 없음)
- [ ] 상충/불확실 사실을 완곡하게 표현했는가
- [ ] 본문에 URL을 나열하지 않았는가
- [ ] SEO 금지사항(도배, 복붙, 낚시제목 등)을 어기지 않았는가
