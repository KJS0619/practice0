# Style Guide — 말투/문체/패턴 분석

`말투1~4.txt` 4개 글을 분석한 결과, 하나의 통일된 말투가 아니라 **서로 다른 3가지 글쓰기 유형**이 확인되었습니다. 각 유형을 구분해서 정리하고, 마지막에 공통적으로 발견되는 패턴을 정리했습니다.

---

## 유형 A — 감성적 독백/일기체 (말투1)

내면의 감정을 쏟아내는 1인칭 독백 스타일입니다.

**문장 구조**
- 문장이 짧고, 한 줄에 한 문장 + 빈 줄로 호흡을 끊음 (시처럼 배치)
- 완결되지 않고 말줄임표(`..`)로 끝나는 문장이 많음 → 여운, 미완의 감정
- 의문형 종결이 압도적: "~일까?", "~것일까?", "~인지." 로 끝나는 자문(自問)이 반복됨

**어미/어투**
- 반말체 독백 (해라체 X, 혼잣말체 O): "무섭고 두렵다.", "물어보고 싶다."
- 감정 어휘 직접 노출: 분노, 슬픔, 충격, 무섭다, 두렵다, 괴롭다
- 논리적 결론 없이 질문으로 문단이 끝나는 패턴 반복 (답을 찾기보다 감정을 토로)

**패턴**
- "왜 나만 그런것일까?" 식으로 억울함·부당함의 정서를 반복 제시
- 자기 권리·정당성에 대한 확인 욕구가 반복 소재로 등장

---

## 유형 B — 기술 튜토리얼/블로그체 (말투2, 말투3)

C언어 학습 내용을 정리하는 티스토리 블로그 포스트입니다.

**구조 패턴**
- **영→한 대조 번역 구조**: 영어 원문 문단 → 그 아래 한국어 번역/설명 문단이 쌍으로 반복됨
- 코드 블록(````struct```` 등)이 설명 사이사이에 삽입됨
- 소제목(Structures, Typedefs, Dereferencing 등)으로 섹션을 나누고, 끝에 `Exercise`(연습문제) + 예시 코드로 마무리하는 정형화된 틀
- 블로그 UI 텍스트(카테고리, 검색하기, 좋아요공감 등)가 본문에 그대로 포함 — 티스토리 특유의 레이아웃 잔재

**어미/어투**
- 설명체 종결: "~이다", "~한다", "~된다" (평서체, 객관적 서술)
- 번역투 문장이 다소 남아있음 (직역 느낌: "정의이보자", "진짜 편리" 등 축약/구어 혼용)
- 한국어 문장 속에 영어 용어를 그대로 노출 (Structures, Typedefs, pointer, Serialization 등) — 번역하지 않고 병기

**패턴**
- 개념 정의 → 이유(왜 쓰는지, 목적) → 코드 예시 → 한 줄 요약 번역의 흐름이 반복
- "~을/를 위해 사용된다", "~할 때 좋다/편리하다" 같은 효용 중심 서술

## 유형 C — 전문 칼럼/에세이체 (말투4, 네이버랩스 Forward Thinking)

AI/로봇공학 전문가의 대중 교양 칼럼입니다.

**문장 구조**
- 문장이 길고 복문 구조 (종속절 + 예시 + 결론이 한 문장에 압축)
- 문단 시작에 독자를 끌어들이는 **수사적 질문**을 배치: "알파고가 이세돌 9단을 이겼을 때 어떠셨나요?", "무엇을 드셨는지 기억나시나요?"
- 구체적 사례(보스턴 다이나믹스, 알파고, 왓슨, 오픈AI 루빅스 큐브 등)를 들어 추상적 주장을 뒷받침
- 문단 중간에 **인용구/명언**을 독립된 짧은 줄바꿈 블록으로 강조 (데이비드 흄 인용 등), 핵심 메시지도 같은 방식으로 하이라이트 처리

**어미/어투**
- 정중한 습니다체 (공식 강연/기고문 톤): "~것입니다", "~습니다", "~하겠죠"
- "~일 것입니다", "~일 수도 있습니다" 등 단정을 피하는 완곡한 추측 표현이 잦음
- 전문용어를 쓰되 바로 쉬운 비유로 풀어줌 (목적 함수 → 방 청소 예시로 설명)

**패턴**
- Part 01/02/03 구조로 논지를 단계적으로 전개 (도입 질문 → 사례 → 통찰 문장으로 문단 마무리)
- 각 Part 말미에 그 파트의 결론을 짧은 두 줄짜리 "카드형 문장"으로 요약 (예: "AI는 최적의 해결책을 찾아낸다. / 반면 사람은 충분히 좋은 방법들을 빠르게 찾는 것에 능숙하다.")
- 필자·삽화가 소개, 출처, 시리즈 소개가 말미에 정형화되어 붙음

## 공통적으로 관찰되는 패턴 (전체 4편 종합)

1. **의문형을 사고 전개 장치로 사용**: 4편 모두 독백형이든 칼럼형이든 "~까?", "~것일까?" 류의 질문을 던지고 그에 답하거나 답을 유보하는 구조를 즐겨 씀. (유형A는 답 없이 감정으로, 유형C는 사례로 답함)
2. **구체 사례/비유로 추상을 설명**: 유형B는 코드 예시, 유형C는 실생활 비유(청소, 치타, 혀의 움직임)로 개념을 구체화.
3. **문단 말미에 요약/통찰 한 줄을 배치**하는 습관 — 유형A는 감정 문장, 유형C는 명제형 요약 문장으로 문단을 맺음.
4. **한/영 병기 혹은 외래어 그대로 사용**을 주저하지 않음 (유형B가 가장 강하지만 유형C도 전문용어 원어 병기: Anthropomorphism, objective function 등).
5. **긴 서술보다 짧게 끝맺는 문장 선호** (특히 유형A, 유형C의 요약구절) — 만연체보다 툭 끊어내는 종결이 강조 효과로 쓰임.

---

## 활용 가이드 (앞으로 이 말투로 글을 쓸 때)

- **감성 에세이/일기**를 쓸 때 → 유형A: 짧은 문장 + 줄바꿈 + 자문형 종결 + 결론을 내리지 않고 감정으로 마무리.
- **기술 설명/튜토리얼**을 쓸 때 → 유형B: 개념→이유→코드/예시→한줄 정리 흐름, 전문용어는 원어 그대로 병기.
- **교양 칼럼/인사이트 글**을 쓸 때 → 유형C: 질문으로 열기 → 구체 사례 → 비유로 풀기 → 문단 끝에 통찰 요약 문장, 큰 주제는 Part로 나누고 각 Part 끝에 카드형 두줄 요약.
